# 校园寝室自助签到系统 - 用户意图功能分析

## 一、功能背景与目标

用户意图识别是本系统的核心功能。通过大语言模型（LLM）分析学生的自然语言输入，识别其真实意图，并路由到对应的功能模块处理。

**目标**：实现精准的意图分类和无缝的多模块调度。

---

## 二、意图分类体系

### 意图类别定义

| 编号 | 意图名称 | 触发关键词 | 处理模块 | 优先级 |
|------|---------|-----------|---------|--------|
| 1 | 签到打卡 | 签到、打卡、报到、我在寝室 | CheckInAssistant | 高 |
| 2 | 请假报备 | 请假、不回来、晚归、病假、回家 | LeaveRequestAssistant | 高 |
| 3 | 状态查询 | 签到记录、请假状态、我的考勤 | QueryCheckInAssistant | 中 |
| 4 | 规则问答 | 几点关门、违纪标准、请假流程 | RAG 语义检索 | 中 |
| 5 | 其他/闲聊 | 你好、谢谢、无关话题 | 直接回复 | 低 |

## 三、意图识别流程

```
用户输入消息
    │
    ▼
┌─────────────────────┐
│  加载对话历史上下文   │
│  (ChatMemory + DB)  │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  构建意图识别Prompt  │
│  历史消息 + 当前输入  │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  调用 LLM 分析意图   │
│  (qwen-max)         │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  解析 JSON 输出      │
│  {intention, output} │
└──────────┬──────────┘
           │
    ┌──────┼──────┬──────┬──────┐
    ▼      ▼      ▼      ▼      ▼
  [1]    [2]    [3]    [4]    [5]
  签到   请假   查询   规则   闲聊
```

## 四、意图识别提示词设计

### 设计原则
1. **角色明确**：定义为"意图识别专员"
2. **分类清晰**：5个类别有明确的边界定义
3. **示例驱动**：每个类别配1-2个示例
4. **严格输出**：要求JSON格式，便于程序解析
5. **上下文感知**：结合历史对话判断意图

### 提示词模板
```
你是一个校园寝室自助签到系统的意图识别专员。
你的唯一任务是分析用户输入的意图，并输出严格的JSON格式。

## 可用意图类别
1 - 签到打卡：学生要进行寝室签到
2 - 请假报备：学生要请假或申请离校
3 - 状态查询：学生查询签到记录或请假状态
4 - 规则问答：学生询问寝室管理规定
5 - 其他/闲聊：不属于以上类别

## 输出格式
{"intention": 编号, "output": "简要描述"}
```

## 五、多轮对话上下文管理

### 上下文来源
1. **ChatMemory**：最近20条消息（内存缓存）
2. **chat_history**：数据库中的完整历史记录

### 上下文应用场景
- 学生说"我要签到" → 意图=1 → 进入签到流程
- 学生继续说"20240101" → 结合上下文知道这是学号
- 学生继续说"张三" → 结合上下文知道这是姓名
- ...直到所有信息收集完毕

---

## 六、异常处理

| 场景 | 处理方式 |
|------|---------|
| LLM 返回非法 JSON | 使用正则提取，兜底意图=5 |
| 意图模糊（如"帮我处理一下"） | 优先询问澄清 |
| 多意图混合 | 识别主意图，提示还有其他意图 |
| LLM 超时/失败 | 返回默认回复，提示重试 |
