# 校园寝室自助签到系统 - 功能模块分析文档

## 一、系统概述

本系统是一个基于 **LangChain4j + Spring Boot** 的 AI 智能寝室签到助手。学生通过自然语言对话完成寝室签到、请假报备、考勤查询等操作，无需手动填写表单。

---

## 二、核心功能模块划分

### 1. AOP 日志拦截模块 (AOP Aspect)
- **职责**：拦截学生端的用户输入请求，进行异步解耦处理
- **功能**：
  - 拦截带 `@ChatFlow` 注解的方法
  - 将学生消息和 AI 回复持久化到 `chat_history` 表
  - 支持辅导员后续审计和对话溯源

### 2. AI 大模型服务模块 (AI Service & Intent Routing)
- **职责**：意图识别与路由分发
- **功能**：
  - **意图识别**：结合"查寝助手" Prompt 分析学生输入意图
  - **上下文管理**：管理 ChatMemory（20条消息窗口）+ 查询历史对话
  - **路由分发**：根据意图编号（1-5）分发到对应处理模块

### 3. 多轮对话登记模块 (Multi-turn Registration Service)
- **职责**：驱动多轮对话采集完整信息
- **功能**：
  - **签到打卡流**：采集学号、姓名、寝室号、楼栋、位置信息
  - **请假报备流**：采集请假类型、起止时间、详细原因
  - 判断信息是否完整，未完整则继续引导补充

### 4. 混合检索与查询模块 (Hybrid Query Service)
- **职责**：处理查询类请求
- **功能**：
  - **签到/请假状态查询**：调用工具查询数据库
  - **规则问答 (RAG)**：向量化寝室管理条例，语义检索回答问题

---

## 三、意图分类体系

| 编号 | 意图 | 触发示例 | 处理模块 |
|------|------|---------|---------|
| 1 | 签到打卡 | "我要签到"、"打卡" | CheckInAssistant |
| 2 | 请假报备 | "我要请假"、"今晚不回来" | LeaveRequestAssistant |
| 3 | 状态查询 | "我的签到记录"、"请假审批了吗" | QueryCheckInAssistant |
| 4 | 规则问答 | "几点关寝室门"、"请假怎么审批" | RAG 语义检索 |
| 5 | 其他/闲聊 | "你好"、"谢谢" | 直接回复 |

## 五、AOP 工作流

```
用户输入 → @ChatFlow AOP拦截
    ├── 旁路任务: 保存消息到 chat_history
    └── 主干任务: AI 意图识别
                    ├── 意图1: 签到服务 → 多轮对话 → 完成后写 check_in_record
                    ├── 意图2: 请假服务 → 多轮对话 → 完成后写 leave_request
                    ├── 意图3: 查询工具 → 读取数据库 → 返回结果
                    ├── 意图4: RAG检索 → 向量匹配 → 返回答案
                    └── 意图5: 兜底回复
```

## 六、技术架构

| 层级 | 技术选型 |
|------|---------|
| 后端框架 | Spring Boot 3.4.4 |
| AI 框架 | LangChain4j 1.0.1-beta6 |
| 大模型 | 阿里云 DashScope (qwen-max) |
| 嵌入模型 | text-embedding-v3 |
| ORM | Spring Data JPA + MyBatis-Plus |
| 数据库 | MySQL 8.x |
| 流式传输 | Spring WebFlux (SSE) |
| 前端 | Thymeleaf + 粒子动画 Chat UI |
